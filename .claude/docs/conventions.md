- [ ] 실패 시 정책이 확정되었는가 (reject / retry / fallback / fail-close)

## Fail-close 원칙

```
// ❌ 금지 — 조용한 보정
const value = parsed || defaultValue;

// ✅ 필수 — 명시적 에러
if (!parsed) throw new Error('파싱 실패: [이유]');
```

검증 실패·파싱 실패·외부 데이터 이상 시 조용히 보정하지 않는다.
