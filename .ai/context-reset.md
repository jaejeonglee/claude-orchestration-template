# 컨텍스트 리셋 체크리스트

새 세션 시작, 작업 재개, 또는 컨텍스트 압축(compact) 이후에 실행합니다.

---

## Step 1. 프로젝트 현황 파악

- [ ] `.ai/architecture.md` 읽기 (시스템 구조 재파악)
- [ ] `git log --oneline -10` 실행 (최근 커밋 확인)
- [ ] `git status` 실행 (현재 변경사항 확인)
- [ ] 현재 브랜치 확인

## Step 2. 이어받는 작업 파악

- [ ] `CURRENT_TASK.md` 읽기 (없거나 비어있으면 사람에게 질문)
- [ ] `.ai/docs/TODO.md` 우선순위 항목 확인

## Step 3. 최근 변경 파악

- [ ] `git diff HEAD~3 --name-only` (최근 변경 파일 목록)
- [ ] `.ai/docs/PROJECT.md` 관련 항목 확인 (변경된 파일 기준)

## Step 4. 환경 확인

- [ ] `package.json` 주요 의존성 버전 확인
- [ ] 최근 변경된 스키마가 있다면 관련 문서 확인

## Step 5. 사람에게 현황 보고

보고 형식:
```
현재 브랜치:
마지막 커밋:
CURRENT_TASK 파악한 현재 작업:
이어서 진행할 수 있는 것:
불확실하거나 확인 필요한 것:
```
